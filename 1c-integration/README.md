# Интеграция 1С:Предприятие 8.3 ↔ АСБО

## Архитектура обмена

```
┌──────────────────┐     REST API (JSON)     ┌─────────────────────────────┐
│   АСБО (Web)     │ ──────────────────────→ │  1С:Предприятие 8.3         │
│                  │ ←────────────────────── │                             │
│ React + Express  │   Подтверждение загрузки │  - ОбщийМодуль: Интеграция  │
│ + SQLite         │                         │  - Регл. задание (каждые    │
│                  │                         │    15 мин)                  │
│  Парсинг выписок │  ──────────────────────→ │  - Обработка (ручная)       │
│  Классификация   │  1CClientBankExchange   │                             │
│  Отчёты ДДС      │  (файловый обмен)       │  Создаёт документы:         │
│                  │                         │  - Поступление на р/с       │
└──────────────────┘                         │  - Списание с р/с           │
                                             │  + проводки Дт/Кт           │
                                             └─────────────────────────────┘
```

## Состав файлов

### 1. `ОбщийМодуль_ИнтеграцияАСБО.txt`
Универсальный модуль с базовыми функциями:
- **HTTP-клиент** — вызовы REST API АСБО (`GET /api/payments`)
- **Сопоставление справочников** — поиск контрагента по ИНН, поиск договора, поиск статьи ДДС
- **Создание документов** — `СоздатьПоступление()`, `СоздатьСписание()`
- **Журналирование** — запись результатов обмена в журнал регистрации

**Размещение в конфигурации 1С:**
- Конфигуратор → Общие → Общие модули → добавить `ИнтеграцияАСБО`
- Флаги: Сервер, Вызов сервера

### 2. `РегламентноеЗадание_СинхронизацияВыписок.txt`
Фоновое задание, запускаемое по расписанию.

**Размещение:**
- Конфигуратор → Общие → Регламентные задания → добавить
- Имя метода: `РегламентноеЗадание_СинхронизацияВыписок.ВыполнитьСинхронизацию`
- Расписание: каждые 900 секунд (15 минут)

**Алгоритм работы:**
1. Вызывает `GET /api/payments?status=processed` в АСБО
2. Для каждого платежа:
   - Ищет контрагента по ИНН (создаёт, если нет)
   - Определяет договор и статью ДДС
   - Создаёт документ «Поступление» или «Списание»
   - Проводит документ

### 3. `Обработка_ЗагрузкаИзАСБО.txt`
Интерактивная обработка для ручной загрузки бухгалтером.

**Размещение:**
- Сохранить как внешнюю обработку (`.epf`)
- Или встроить в конфигурацию: Обработки → добавить

**Интерфейс:**
- Выбор периода (дата с/по)
- Таблица предпросмотра с колонками:
  - ☐ Выбран | Номер | Дата | Контрагент | ИНН | Сумма | Тип | Статья ДДС | Назначение | Статус
- Кнопки: «Обновить список», «Загрузить всё», «Загрузить выбранные»

## Требуемые справочники и документы в типовой конфигурации 1С

| Объект 1С | Тип | Назначение |
|---|---|---|
| `Организации` | Справочник | ООО «Социальные услуги» (ИНН 7713699602) |
| `Контрагенты` | Справочник | Плательщики и получатели из выписок |
| `ДоговорыКонтрагентов` | Справочник | Привязка платежа к договору |
| `СтатьиДвиженияДенежныхСредств` | Справочник | Классификация ДДС |
| `ПоступлениеНаРасчетныйСчет` | Документ | Приходные платежи |
| `СписаниеСРасчетногоСчета` | Документ | Расходные платежи |

## Порядок работы (типовой сценарий)

1. Бухгалтер загружает выписку из интернет-банка в АСБО
2. АСБО парсит файл, классифицирует платежи, сопоставляет контрагентов и статьи ДДС
3. **Вариант А (авто):** Регламентное задание 1С каждые 15 минут забирает готовые платежи и создаёт документы
4. **Вариант Б (ручной):** Бухгалтер открывает обработку «Загрузка из АСБО», проверяет список, нажимает «Загрузить»
5. В 1С появляются проведённые документы с заполненными реквизитами и сформированными проводками Дт 51 / Кт 62 (или Дт 60 / Кт 51)
6. Статья ДДС, определённая в АСБО, проставляется в документе для последующего формирования отчёта о движении денежных средств в 1С

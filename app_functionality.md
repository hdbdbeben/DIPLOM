# АСБО — Автоматизированная система банковских операций

## Краткое описание функционала

Веб-приложение для ООО «Социальные услуги», автоматизирующее учёт банковских операций: загрузку выписок, классификацию платежей, ведение справочников, формирование отчёта ДДС, обработку ошибок и интеграцию с 1С:Предприятие.

### Технологический стек

| Слой | Технологии |
|---|---|
| Фронтенд | React 18, TypeScript, Vite 6, React Router 6 |
| Бэкенд | Express 4, TypeScript |
| База данных | SQLite (better-sqlite3); целевая production — MySQL |
| API-моки (dev) | MSW 2 (Mock Service Worker) — перехват на уровне Service Worker |
| Парсер выписок | 1CClientBankExchange v1.03, TXT (CSV), XML |

### Роли пользователей

| Роль | Доступ |
|---|---|
| Администратор (`admin`/`admin`) | Полный доступ: все разделы + администрирование + журнал действий |
| Бухгалтер (`buh`/`buh123`) | Работа с выписками, платежами, справочниками, отчётами, ошибками, 1С |
| Руководитель (`dir`/`dir123`) | Просмотр отчётов ДДС, выписок, платежей (без редактирования) |

### Функциональные разделы (10 страниц)

#### 1. Авторизация (`/login`)
- Вход по логину/паролю с выбором роли
- Отображение демо-доступа

#### 2. Банковские выписки (`/statements`)
- Загрузка файлов выписок в форматах: 1CClientBankExchange, TXT (CSV), XML
- Автоматический парсинг с определением формата
- Загрузка демо-данных для тестирования
- Просмотр архива загруженных выписок со статистикой (количество операций, статус)
- Экспорт выписки в формат 1CClientBankExchange
- Удаление выписок
- Переход к списку платежей конкретной выписки

#### 3. Платёжные операции (`/payments`)
- Таблица всех платёжных операций с пагинацией
- Фильтрация: по статусу, текстовый поиск (с debounce 400 мс), по ID выписки
- Детальный просмотр операции в модальном окне с формой классификации:
  - Выбор контрагента (клиента)
  - Выбор типа платежа
  - Выбор статьи ДДС (фильтруется по типу: доход/расход)
  - Изменение статуса
- Массовое выделение операций
- Отправка платёжных поручений в банк:
  - Формирование и скачивание файла
  - Отправка по FTP
- Создание нового платежа вручную (`/payments/new`):
  - Форма с выбором плательщика, получателя, договора, статьи
  - Автозаполнение ИНН и расчётного счёта при выборе контрагента

#### 4. Справочники (`/directories`)
Пять вкладок с полным CRUD-управлением:
- **Контрагенты** — поиск по наименованию/ИНН, реквизиты
- **Банки** — БИК, корр. счёт, наименование
- **Типы платежей** — код, наименование
- **Статьи ДДС** — тип (доход/расход), код, наименование
- **Договоры** — привязка к контрагенту, номер, дата

Все операции через модальные формы, удаление с подтверждением.

#### 5. Отчёты (`/reports`)
- **Отчёт о движении денежных средств (ДДС)**:
  - Выбор диапазона дат (по умолчанию — текущий год)
  - Сводная таблица по статьям с разбивкой: доходы, расходы
  - Итоговые строки: всего доходов, всего расходов, чистый поток
  - Экспорт в CSV (UTF-8 BOM для совместимости с Excel)

#### 6. Журнал ошибок (`/errors`)
- Список исключений при обработке выписок
- Фильтрация по статусу: новая, в работе, решена
- Управление ошибкой:
  - Взять в работу
  - Назначить ответственного (из списка бухгалтеров/администраторов)
  - Отметить как решённую

#### 7. Интеграция с 1С (`/1c`)
Три вкладки:
- **Импорт из 1С** — просмотр организаций и договоров из 1С, массовый импорт выбранных договоров (автосоздание контрагентов)
- **Экспорт в 1С** — список платежей, готовых к экспорту; массовый экспорт с отслеживанием статуса каждого документа
- **Журнал обмена** — хронология всех операций импорта/экспорта с указанием статуса и направления

#### 8. Журнал действий (`/logs`)
- Хронологическая таблица всех действий пользователей
- Записи о загрузке выписок, изменении статусов ошибок, экспорте
- Доступ только для администратора

#### 9. Администрирование (`/admin`)
Три вкладки:
- **Пользователи** — CRUD с назначением роли, активация/деактивация
- **Роли** — просмотр справочника ролей
- **Настройки** — резервное копирование БД, восстановление из файла, полный сброс данных с двойным подтверждением

### Ключевые технические особенности

- **Парсер выписок** — автоматическое определение формата, поддержка 1CClientBankExchange v1.03, TXT с разделителями, XML
- **API-клиент** — типизированный HTTP-клиент с обработкой ошибок и поддержкой blob-ответов
- **Моки (MSW)** — 30+ эндпоинтов, 100+ записей для разработки и демонстрации без бэкенда
- **UI-компоненты** — модальные окна, тосты, табы, статусные бейджи, спиннеры
- **Безопасность** — хеширование паролей, разграничение доступа на уровне роутов и API
